# Parcours

## A qui s'adresse ce contenu

Ce contenu vise des developpeurs qui utilisent deja des assistants IA ou qui vont commencer a les integrer dans leur travail. Le niveau technique logiciel peut etre bon, mais le vocabulaire IA n'est pas toujours stabilise. L'objectif est donc de fournir des reperes simples sans perdre en precision.

## Ce que vous devez etre capable de faire apres lecture

Apres ce parcours, vous devriez pouvoir :

1. expliquer la difference entre IA, IA generative, modele et LLM ;
2. decrire ce qu'est le contexte et pourquoi sa qualite conditionne les resultats ;
3. distinguer un prompt, une instruction, un workflow, un outil et un agent ;
4. comprendre le role d'un RAG, d'une memoire ou d'un protocole comme MCP ;
5. utiliser des outils de type Copilot ou Claude Code avec un meilleur niveau de controle.

## Ordre de lecture

### 1. Fondamentaux

Le chapitre [Fondamentaux](01-fondamentaux.md) fixe le vocabulaire. C'est la base a partager quand on parle d'IA dans une equipe.

### 2. Fonctionnement agentic

Le chapitre [Fonctionnement agentic](02-fonctionnement-agentic.md) explique la mecanique reelle des assistants et des agents : contexte, outils, instructions, boucle d'action, limites.

### 3. Outils et ecosysteme

Le chapitre [Outils et ecosysteme](03-outils-ecosysteme.md) relie les concepts aux outils que les developpeurs rencontrent, comme GitHub Copilot, Claude Code, les fichiers de configuration, les workflows et MCP.

### 4. Fichiers et conventions

Le chapitre [Fichiers et conventions](05-fichiers-et-conventions.md) detaille la facon dont les equipes rendent explicites leurs regles de travail, notamment via des fichiers places sous `.github` ou `.claude`.

### 5. Pratiques developpeur

Le chapitre [Pratiques developpeur](04-pratiques-developpeur.md) transforme ces notions en methodes de travail utiles pour la production logicielle.

### 6. Ressources transverses

Le [Glossaire](glossaire.md), la [FAQ](faq.md) et les [Idees recues](idees-recues.md) servent de reference rapide. Les [Tableaux comparatifs](comparatifs.md) permettent de mettre cote a cote les notions frequemment confondues.

### 7. Fiches detaillees

Le chapitre [Fiches detaillees](06-fiches-detaillees.md) propose des fiches approfondies sur les notions cles, avec des conventions de fichiers `.github` et `.claude` en exemples concrets.

## Convention de lecture

Chaque notion importante suit la meme structure :

- une definition courte ;
- une explication orientee developpeur ;
- un exemple concret ;
- une section sur les confusions frequentes ;
- une section sur les limites.

## Positionnement du contenu

Ce depot ne cherche pas a trancher quel outil est le meilleur. Il cherche a construire une carte mentale correcte de l'univers agentic pour mieux utiliser les outils disponibles et mieux juger leurs resultats.
